# On the Rename: Substrate to Navigation

**Summary**: Why this wiki changed its central metaphor from "memory as substrate" to "memory as navigation." The rename resolves a lexical tension with Levin's framework, the source the wiki leans on most, and adopts a verb that all four converging sources already use.

**Sources**: Self-Improvising Memory.pdf, self-improvising-memories.pdf, Memory as an Action Space.pdf, Conway_2005.pdf

**Last updated**: 2026-07-10

---

## The problem with "substrate"

The original name, "Memory as Substrate," meant substrate in the sense of a foundation: memory is the layer an agent's identity is built on, not a record stored inside it. That reading is fine and the thesis still holds.

But "substrate" has a second, more common reading in this literature: the storage medium, the physical stuff memory lives in. That is exactly the reading Levin spends his paper arguing against. In his own words there is [no-single-substrate](/no-single-substrate); the synaptic machinery is "mostly for interpretation of the memory, not storage," and biology treats "all the parts" as "a kind of reservoir" (source: self-improvising-memories.pdf).

So the wiki was named after the one noun its central source uses mainly to deny. To anyone arriving via Levin rather than via Conway, the title pointed at the wrong idea. The content was faithful to him; the branding partly contradicted him.

## Two senses of substrate, cutting opposite ways

- **Substrate as storage medium**: the thing memory lives in. Levin rejects this. There is no privileged medium; everything is an interpretable reservoir, and what matters is the agent reading and rewriting it.
- **Substrate as foundation**: what something is built on. The wiki's intended sense. Levin would mostly agree, except he would insist it is a process (interpretation), not a thing, and that no fixed matter performs it.

The old name worked only under the second reading and silently suggested the first.

## Why "navigation"

The rename picks a verb, not a noun, because every converging source lands on activity over stuff:

- **Levin**: memory is an interpretation process you perform across a reservoir. An "interpretable scratchpad" you move across (source: Self-Improvising Memory.pdf).
- **Conway**: memory is reconstructed at every retrieval, modulated by current goals. A construction, not a lookup ([retrieval-models](/retrieval-models)).
- **Youssef**: the shift from retrieval to [ai-memory-as-identity-construction](/ai-memory-as-identity-construction). Memory builds the agent.
- **NapMem**: long-term memory should be "a structured action space" the agent "navigates," not "passively retrieved context" (source: Memory as an Action Space.pdf).

"Navigation" is the one word all four would sign. It keeps Levin's active traversal, Conway's reconstruction-at-retrieval, Youssef's identity work, and NapMem's engineering term.

## What the rename does not change

It resolves a lexical problem, not an empirical one. The substantive disagreement in [open-questions](/open-questions) stays open: Conway places memory construction in specific vertebrate brain networks, Levin denies any single substrate, and the question of whether engineered substrates can support Levin's layer is unresolved. That is a real disagreement about how memory is implemented, distinct from what we call it.

NapMem sharpens that question rather than closing it: it shows the functional layer (hierarchy, goal-filtering, active reinterpretation) delivers most of the gains on a perfectly ordinary substrate, but it does not dissolve the substrate. See [memory-as-action-space](/memory-as-action-space) and [active-memory-navigation](/active-memory-navigation).
