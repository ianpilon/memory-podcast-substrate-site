# Memory as Construction Thesis

**Summary**: The central synthesis of the wiki. Five converging arguments (Conway in psychology, Levin in biology, Youssef, NapMem, and the Proactive Memory Agent in AI engineering) that **memory is not storage to be retrieved; it is the construction process by which an agent persists across time**. The frameworks disagree on substrate-specificity but agree on the function: memory is something you do. Navigation and intervention are the access mechanisms within it.

**Sources**: Conway_2005.pdf, Self-Improvising Memory.pdf, Robert Youssef x post.pdf, Memory as an Action Space.pdf, Proactive Memory Agent.pdf, Polycomputing - Room Right Here.pdf

**Last updated**: 2026-07-10

---

## The thesis in one sentence

Memory is the construction process by which a self (at any scale) maintains identity across change. Treating memory as storage is a category error that explains both why current AI agent memory architectures fail and why the textbook brain metaphors keep getting replaced. See [on-naming](/on-naming) for why this wiki moved through "substrate" and "navigation" to land on "construction."

## Why "construction" sits at the function level

Biological memory and AI memory are not identical and cannot be. They differ at the substrate level: synapses and bioelectric reservoirs in one, weights and JSONL stores in the other. But they can share a *function*: a process that maintains identity across change through reconstruction and goal-gated access. The name has to sit at that shared function level, because it is the only level where the two are genuinely the same thing.

"Construction" is Conway's word and the wiki's own thesis verb. It is a process (so it satisfies the point that memory is a doing, not a stuff), substrate-neutral (you construct with synapses or with weights), distinctive, and broad enough to cover the whole loop (encoding, consolidation, reconstruction, narrative synthesis). Navigation and intervention name the access mechanisms inside that process, not the process itself. See [on-naming](/on-naming).

## The five convergent arguments

### Psychology (Conway)

The [self-memory-system](/self-memory-system) is a bidirectional coupling of [working-self](/working-self) (active goals) and [autobiographical-knowledge-base](/autobiographical-knowledge-base) (hierarchical store). Memory is reconstructed at every retrieval, modulated by current goals, and biased by the [coherence-vs-correspondence](/coherence-vs-correspondence) tradeoff. The function is not archival; it is keeping a goal-pursuing agent operative across time. Retrieval is generative construction, not lookup ([retrieval-models](/retrieval-models)). See [conway-2005-memory-and-self](/conway-2005-memory-and-self).

### Biology (Levin)

[mnemonic-improvisation](/mnemonic-improvisation) is the dynamic ability to rewrite and remap information onto new media and contexts. [memories-as-agents](/memories-as-agents) are active patterns, not passive data. The [bowtie-architecture](/bowtie-architecture) forces creative reinterpretation on the decoder side. There is [no-single-substrate](/no-single-substrate); biology treats every part as an interpretable reservoir. The function is to keep an agent operative across substrate changes (development, regeneration, metamorphosis, lineage). Memory is an interpretation you perform, not a thing you store. See [levin-2024-self-improvising-memory](/levin-2024-self-improvising-memory).

### AI engineering: principles (Youssef)

Current AI agent memory (vector databases, conversation summaries, key-value stores, episodic buffers) fails because it implements storage, not identity. The fix is to apply psychology's principles (hierarchy, goal-filtering, emotional weighting, narrative, co-emergent self-model) and build an identity-construction system. See [youssef-2026-psychology-and-ai-memory](/youssef-2026-psychology-and-ai-memory) and [five-principles-ai-memory-lacks](/five-principles-ai-memory-lacks).

### AI engineering: measurement and intervention (NapMem and the Proactive Memory Agent)

NapMem tests the active-use claim directly. It reframes long-term user memory as [memory-as-action-space](/memory-as-action-space) that the agent [navigates](/active-memory-navigation) with tools, rather than passively retrieved context. Its [memory-pyramid](/memory-pyramid) is an engineering re-derivation of Conway's autobiographical hierarchy. Switching navigation off and reverting to passive retrieval drops accuracy sharply, the strongest measured evidence yet that "memory as storage" underperforms "memory as active use." Crucially, NapMem achieves this on a perfectly ordinary substrate and does not dissolve it (source: Memory as an Action Space.pdf). See [source-napmem](/source-napmem).

The Proactive Memory Agent sharpens the claim one step further. It separates memory from the action agent entirely: a [dedicated memory agent](/proactive-memory-agent) maintains a structured bank and decides whether to inject a reminder or stay silent. Its core reframe is [memory-as-intervention](/memory-as-intervention), and it names the failure mode both prior sources miss: [behavioral-state-decay](/behavioral-state-decay), where decision-relevant state stops influencing the next action even when still in context. Its ablation shows the win lives in [selective-intervention](/selective-intervention), the decision not to interrupt, which beats passive full-bank exposure and always-on injection (source: Proactive Memory Agent.pdf). See [source-proactive-memory](/source-proactive-memory).

### Substrate: is the Levin layer reachable in engineering? (Bongard & Levin)

NapMem and the Proactive Memory Agent prove the functional layer works on an ordinary substrate; they do not test whether genuine substrate flexibility is reachable in engineered systems. That gap is addressed by a third source: Bongard & Levin's polycomputing paper ([source-bongard-levin-polycomputing](/source-bongard-levin-polycomputing)). It defines [polycomputing](/polycomputing) as one substrate simultaneously computing different things for different observers, and reviews engineered systems that already do this (polycomputing metamaterials, physical reservoir computing, holographic storage, memory materials, soft robotics). The honest verdict flips: Levin's layer is reachable in engineered systems, early and un-tooled but real. The remaining open question is not reachability but whether a deployed polycomputing store beats a conventional one on an agent-memory benchmark. See [engineered-substrate-flexibility](/engineered-substrate-flexibility).

## What the five agree on

1. **Memory is functional, not archival.** Its purpose is agent-persistence, not record-keeping.
2. **Reconstruction is the default.** Every retrieval is an active construction, not a lookup.
3. **Memory is something you do.** Construct (Conway) is the umbrella verb; the sources specialize it as interpret (Levin), build identity (Youssef), navigate (NapMem), and intervene (Proactive Memory Agent). Construction names the process; the others name mechanisms within it.
4. **The self and memory are bidirectional.** The agent shapes what is remembered; the memory shapes what the agent can become.
5. **Fidelity is not the optimization target.** Salience, coherence, and identity-relevance are.

## Where they disagree

1. **Substrate specificity.** Conway places SMS construction in specific vertebrate brain networks (fronto-temporal-occipital). Levin denies any single substrate, including synapses. The Schleisman article ([schleisman-promise-of-agi-neuroscience](/schleisman-promise-of-agi-neuroscience)) sits between them: brains have at least 5 features LLMs lack, so neural-net substrate is not sufficient, but it does not commit to substrate-agnostic memory either. NapMem sits on the engineering side of this line: it shows the functional layer works without committing to any substrate claim.
2. **Confabulation framing.** Conway treats correspondence as the default in healthy recent memory; coherence-driven distortion is a regulated process. Levin treats reinterpretation as the default; faithful recall is the special case. See [confabulation-as-feature](/confabulation-as-feature).
3. **Cross-scale generality.** Levin pushes the framework down to cells and up to lineages. Conway is committed to vertebrate-cognitive-level only. Youssef and NapMem apply psychology to engineered AI agents but stay at agent-scale.

## The engineering implication

If the thesis is right, then building useful AI agent memory requires:

- Hierarchy and goal-filtering (Conway and [five-principles-ai-memory-lacks](/five-principles-ai-memory-lacks); instantiated as the [memory-pyramid](/memory-pyramid) in NapMem).
- Active reinterpretation on retrieval, not nearest-neighbor lookup ([bowtie-architecture](/bowtie-architecture), [active-memory-navigation](/active-memory-navigation)).
- Emotional or salience weighting (Conway's working-self conservatism plus Damasio's [somatic-marker-hypothesis](/somatic-marker-hypothesis)).
- A narrative layer that synthesizes memories into an evolving self-model ([narrative-coherence](/narrative-coherence) and [co-emergent-self-model](/co-emergent-self-model)).
- Tolerance for and active use of substrate flexibility (Levin's [polycomputing](/polycomputing) and [no-single-substrate](/no-single-substrate)).

The first four are roughly the [five-principles-ai-memory-lacks](/five-principles-ai-memory-lacks), and NapMem shows the first two deliver measurable gains on their own. The fifth is the Levin layer (substrate flexibility), which the polycomputing source ([source-bongard-levin-polycomputing](/source-bongard-levin-polycomputing)) now shows is reachable in engineered materials via physical reservoir computing and polycomputing metamaterials, even though no programming language or deployed agent-memory benchmark exists yet. See [open-questions](/open-questions) for what stays unresolved.

## Reconciling biological and AI memory

The thesis is deliberately function-level so it can hold for both biological and engineered agents without claiming they are identical. The honest reconciliation: biological memory and AI memory are functionally homologous (both are construction processes that maintain identity across change) and materially different (they are built on different substrates, with different fidelity, different forgetting dynamics, and different ceilings on the Levin layer). The name lives at the shared function; the substrate difference is stated explicitly wherever it matters, which is what [open-questions](/open-questions) is for.
