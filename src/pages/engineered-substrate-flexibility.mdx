# Engineered Substrate Flexibility

**Summary**: The engineering answer to this wiki's substrate-flexibility open question. Levin's [no-single-substrate](/no-single-substrate) layer is not a biological curiosity; it is a property already present in engineered materials, and overloading multiple functions onto the same hardware is an explicit design principle. This page catalogs the engineered analogues and states the revised verdict: the Levin layer is reachable in engineered systems, early and un-tooled but real.

**Sources**: Polycomputing - Room Right Here.pdf

**Last updated**: 2026-07-10

---

## The open question this page closes

[open-questions](/open-questions) section 3 asks: is Levin's layer (genuine substrate flexibility, the ability of one substrate to be reinterpreted as carrying different memories or computations by different observers) reachable at all in engineered systems, or is it a biological property that silicon and software can only imitate at the functional level? Until this source, the wiki rested the claim on Levin's memory paper, where it is asserted from invertebrate evidence.

The polycomputing paper ([source-bongard-levin-polycomputing](/source-bongard-levin-polycomputing)) changes the status. It reviews engineered systems that already compute and store multiple results in the same locality at the same time, and argues the property is a design target, not a biological monopoly. The honest revised verdict: **the Levin layer is reachable in engineered systems. It is early, it has no programming language yet, and no deployed agent-memory benchmark proves it beats a conventional store, but it is real and under active construction.**

## The four engineered analogues

### 1. Non-quantum polycomputing metamaterials

Bongard showed that a metamaterial of roughly 30 parts can compute two Boolean functions (AND and XOR) in the same place at the same time, without any quantum effect. This kills the assumption that superposed computation requires quantum mechanics. Different observers reading the same material extract different logical results. This is the minimal engineered demonstration that one substrate can carry multiple simultaneous computations (source: Polycomputing - Room Right Here.pdf, section 3).

### 2. Holographic data storage

Data is etched as a distributed representation across the storage medium rather than at a localized address, and retrieved by illuminating the medium from a particular direction. The same physical locality holds fragments of many different data, which are accessed at different times by probing from different angles. Storage density is decoupled from spatial address. This is the closest engineered analogue to Levin's claim that memory is distributed across a reservoir rather than filed in a slot (source: section 3).

### 3. Physical reservoir computing

This is the most direct engineered instantiation of Levin's reservoir-computing framing in [no-single-substrate](/no-single-substrate). An inert bulk material (a metal plate, a photonic crystal, a bucket of nonlinear medium) is excited with forces or light, and the resulting vibrations or refracted patterns are read out as the result of a computation. The material's internal structure is never changed during training; the "weights" live in how you drive and read it. 

The paper highlights "deep physical neural networks," where the input and the network's parameters are both encoded as forces applied to the material, and the material's response is read back as the network output. Training modulates the input forces until the output is correct. Because the material itself is unchanged, **the same material can embody different neural networks**. This is engineered substrate flexibility in literal form: one substrate, many computations, selected by the read/write interface rather than by rewiring the medium (source: section 3).

### 4. Memory materials and soft robotics

Some materials are not only changed by the forces acting on them but retain an imprint of those forces after they cease. Active engineering effort aims to design such materials to maximize the number of overlapping memories they can store in the same place. In soft robotics the same body parts already simultaneously perform sensing, actuation, computation, and energy storage, dissolving the boundary between the behaviour-generating body and the computation-generating brain (source: section 3).

## Why the read-side is the lever

Section 4.4 of the paper contains the cleanest operational statement of how engineered substrate flexibility actually works, and it maps directly onto this wiki's [polycomputing](/polycomputing) read/write distinction. Biological gene regulatory networks, probed as Pavlovian learning circuits, turn out to have latent memory capacity. The key finding: **the same network offers multiple different memory and computation types depending on which nodes the observer chooses as control knobs and readouts.**

The implication for engineering is that the path to substrate flexibility is not always to rewire the medium but to find the perspective from which an unmodified system already embodies the function you want. This is the read-side reinterpretation that [mnemonic-improvisation](/mnemonic-improvisation) and [bowtie-architecture](/bowtie-architecture) describe biologically, now stated as an engineering search problem: given a substrate, find the interface that exposes the computation you need without disturbing the others.

## The catastrophic-interference hook

The paper names catastrophic interference (catastrophic forgetting) as a problem polycomputing may solve. A conventional agent overwrites old learning with new because both compete for the same representational real estate. A polycomputing agent could store a new behaviour at an underutilized place on its substrate's frequency spectrum, leaving the existing behaviour intact.

This matters for the identity-persistence thesis ([memory-as-construction-thesis](/memory-as-construction-thesis)) because catastrophic forgetting is precisely the failure mode that makes an agent lose identity over time on a conventional substrate. If polycomputing substrates can suppress it, they are candidate substrates for long-horizon identity-coherent agents, which is the regime where [open-questions](/open-questions) section 3 says the substrate floor might reassert itself.

## The honest ceiling

Three limits keep this from being a solved engineering problem:

1. **No programming language.** The paper explicitly states no programming language has yet been devised for polycomputational metamaterials. The tooling to specify a memory on such a substrate does not exist.
2. **No agent-memory benchmark.** The evidence is reachability and analogy, not a deployed system beating a vector store or JSONL bank on an agent task. The measured engineering gains the wiki has so far ([source-napmem](/source-napmem), [source-proactive-memory](/source-proactive-memory)) come from the functional layer on ordinary substrates, not from polycomputing hardware.
3. **It does not dissolve Conway's substrate.** Conway's vertebrate neuroanatomy claims ([conway-2005-memory-and-self](/conway-2005-memory-and-self)) are about a specific biological implementation. Engineered polycomputing shows flexibility is reachable; it does not show the flexible engineered substrate is the same kind of thing as a hippocampal network, only that both can in principle support reinterpretation.

The revised open question is therefore not "is the Levin layer reachable in engineered systems" (yes) but "does anything in current engineered practice actually reach it well enough to matter for agent memory, and does a substrate advantage appear at long horizons." That stays open. See [open-questions](/open-questions).
